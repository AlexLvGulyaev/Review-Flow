## Что увидел на `cursor_error.png` (текущий результат)

- **Композиция не “widescreen”**: контент визуально сжат в центральную колонку, а вокруг — большие пустые поля.
- **Большая пустота перед footer**: между блоком “О Northline Market” и footer заметен огромный пустой серый участок.
- **Hero выглядит как отдельная карточка/баннер**: нет ощущения цельной фото‑секции как в reference.
- **Плотность блоков ниже hero ниже reference**: strip/cards выглядят более “мелкими” и оторванными.

## Чем отличается от `ui_reference_client_homepage_widescreen_v2.png`

- В reference страница воспринимается как **единая композиция** на 16:9: header → hero → strip → cards → info → footer.
- В текущем скрине важные зоны “теряются” из-за **пустот** и поведения footer (визуально “уезжает” вниз).

## Differences: current vs reference (по зонам)

### 3.1 Общая высота страницы

- **Почему стала “слишком высокой/пустой”**: пустота создаётся не столько высотой блоков, сколько layout‑поведением оболочки.
- **Ключевой источник**: `.client-shell { display:flex; flex-direction:column; }` + `.client-footer-rich { margin-top:auto; }` → footer прижимается вниз окна, а при недостатке контента остаётся “пустая полоса”.

### 3.2 Header

- **Reference**: компактный, в одну линию, визуально “интегрирован” с hero.
- **Current**: структура правильная, но визуально может казаться маленьким из-за узкого `--client-max` и общего масштаба композиции.

### 3.3 Hero

- **Reference**: светлый “фото‑фон”, умеренная высота, композиция читается как маркет‑секция.
- **Current**: на скрине hero выглядит как отдельный баннер/карточка; вероятная причина — несоответствие фон/радиусов/теней или устаревший билд на момент скрина.

### 3.4 Product illustration

- **Reference**: фотореалистичная композиция (коробка+растение+пакет), мягкая тень.
- **Current**: иллюстрация стилизована/векторная; масштаб и “материал” нужно подгонять под reference.

### 3.5 Feature strip под hero

- **Reference**: 5 элементов, очень плотные, небольшой vertical footprint.
- **Current**: элементы присутствуют, но суммарная высота/отступы выглядят более рыхлыми.

### 3.6 Большие feature cards

- **Reference**: 4 карточки, умеренная высота, плотный текст.
- **Current**: карточки визуально легче и могут быть выше/реже по тексту.

### 3.7 Company/info block

- **Reference**: “фото” слева, текст и KPI справа, плотный блок.
- **Current**: структура есть, но блок + пустота до footer ломают “единый экран”.

### 3.8 Footer

- **Reference**: footer следует сразу за контентом.
- **Current**: footer “оторван” из-за `margin-top:auto`.

## Correction plan (конкретные правки)

> В рамках Task 017B это **план**, не применение.

### План A (первый минимальный fix — убрать пустоту)

- **Footer**: удалить `margin-top: auto;` у `.client-footer-rich`.
  - Сейчас: `.client-footer-rich { margin-top: auto; ... }`
  - Будет: footer идёт сразу после контента, без “пустой ленты”.

### План B (подгонка плотности к reference после A)

- **Общая ширина**: увеличить `--client-max` с `1200px` до `1320px` (или `1360px`) чтобы уменьшить ощущение узкой центральной колонны на 1920px.
- **Hero**:
  - уменьшить `.client-hero-reference-inner` `min-height` с `420px` до `340–360px`;
  - уменьшить `padding` hero сверху/снизу (например `2.6rem` → `2.1rem`, низ `2.2rem` → `1.6rem`);
  - ограничить/сдвинуть иллюстрацию: `max-width` + возможный `transform: translateX(...)` без роста высоты.
- **Feature strip**:
  - `.client-features-inner` `padding: 1.1rem` → `0.85rem`;
  - `.client-features-inner gap: 0.9rem` → `0.75rem`;
  - `.client-feature-item padding: 0.75rem` → `0.6rem`;
  - иконки 40px → 36px.
- **Большие cards**:
  - `.client-wide-card padding: 1rem` → `0.85rem`;
  - `h3` и `p` шрифты на -1–2px; line-height плотнее.
- **Info block**:
  - ограничить высоту `.client-about-image` через `min-height`/`aspect-ratio`;
  - KPI карточки: padding на -2–4px, gap меньше.
- **Секции**:
  - `.client-section { margin-top: 2.5rem; }` → `1.75–2rem` (приближение к reference плотности).

## Риски исправления

- Удаление “липкого” footer меняет поведение на коротких страницах (footer перестанет прижиматься вниз окна) — но это ближе к reference.
- Увеличение `--client-max` может визуально затронуть другие client‑страницы — лучше ограничить изменения homepage‑классами (или отдельным контейнером).
- Слишком сильное уменьшение отступов ухудшит читаемость на 1366×768 — надо проверять как минимум 1440×900 и 1920×1080.
